# Features

## 增量更新
核心思路：**Diff-based** 增量更新方式。配置文件更新时，先对比新旧配置 (diff)，只更新需要变更的配置项。(思路参考 React 的 VDOM)

### 实现架构

入口方法：`GatewayState::diff_update(&mut self, new_config)` — 持有 `config_snapshot` 快照，与 `new_config` 对比后按模块 diff。

```text
diff_update(new_config)
  ├── diff_routes(old, new)        — 路由增删改
  ├── diff_clusters(old, new)      — 上游集群增删改
  ├── diff_rate_limiter(old, new)  — 限流器策略调整
  ├── diff_auth(old, new)          — 认证配置更新
  └── config_snapshot = new        — 更新快照
```

### 各模块 Diff 策略

#### 1. 路由 Diff（`diff_routes`）
- 以 `route_id` 作为唯一标识
- **删除**：旧有新无的 `route_id`，调用 `router.remove_route()` + `registry.unregister()`
- **新增**：新有旧无的 `route_id`，调用 `add_route_from_config()`
- **变更**：`route_id` 相同但内容不同（`routes_differ()` 比较 path/prefix/match_type/methods/upstream/is_auth），先删后增

#### 2. 上游集群 Diff（`diff_clusters`）
- 以服务名称作为唯一标识
- **删除**：旧有新无的服务，从 `clusters` HashMap 移除
- **新增/变更**：调用 `UpstreamCluster::from_config()` 重建
- **未变更的集群保持原实例**（Arc 指针不变），保留健康检查状态和连接池

#### 3. 限流器 Diff（`diff_rate_limiter`）
- **无→有**：创建新 `RateLimiter`
- **有→无**：移除限流器
- **参数变更**：调用 `update_policy()` 保留令牌桶状态，仅更新 capacity/refill_rate
- **未变更**：不做操作

#### 4. 认证 Diff（`diff_auth`）
- 比较 algorithm/public_key_path/issuer/claims_to_forward
- **变更时**：重新从 PEM 文件加载公钥，构建新 `AuthConfig`

### 触发机制
`ControlPlane` 通过文件监听（`notify` crate）检测 `config.yaml` 变更，读取新配置后调用 `state.write().unwrap().diff_update(&new_config)` 完成热重载。

## 配置项校验
核心思路：**校验与构建逻辑分离**，将校验逻辑实现为纯函数，批量收集所有错误后一次性返回。

### 架构

```text
validate_config(config)                          # 全量校验入口（src/config/validation.rs）
  ├── validate_server(server, errors)             # server 模块校验
  ├── validate_routes(routes, upstreams, errors)  # route 模块校验
  ├── validate_upstreams(upstreams, errors)       # upstream 模块校验
  ├── validate_rate_limit(rl, errors)             # rate_limit 模块校验（可选）
  └── validate_auth(auth, errors)                 # auth 模块校验（可选）

validate_route_config(route, upstreams)           # 单路由校验（供 Admin API 灰度发布复用）
```

- 所有校验函数通过 `&mut Vec<ValidationError>` 批量收集错误，不遇错即停
- `parse_socket_addr()` 作为私有工具函数，复用于 server.listen 和 upstream node.addr 的地址格式校验
- 校验结果通过 `StateError::ConfigValidationFailed` 返回给调用方

### 集成点

| 调用位置                          | 说明                                   |
|-------------------------------|--------------------------------------|
| `GatewayState::from_config()` | 构建前先校验，失败返回 `ConfigValidationFailed` |
| `GatewayState::diff_update()` | 热重载前先校验新配置，校验不通过则不应用变更               |

### 配置校验项列表

| 类别         | 字段              | 校验规则                      |
|------------|-----------------|---------------------------|
| server     | listen          | `ip:port` 格式，端口 0-65535   |
| server     | threads         | 若存在必须 > 0                 |
| route      | route_id        | 非空、唯一                     |
| route      | match_type      | 枚举：exact / prefix / regex |
| route      | path            | exact/regex 时必须存在且非空      |
| route      | path_prefix     | prefix 时必须存在且非空           |
| route      | upstream        | 必须引用已存在的 upstream         |
| upstream   | nodes           | 至少一个节点                    |
| upstream   | node.addr       | `ip:port` 格式              |
| upstream   | node.weight     | `> 0`                     |
| rate_limit | capacity        | `> 0`                     |
| rate_limit | refill_rate     | `> 0`                     |
| auth       | algorithm       | 当前仅支持 `RS256`             |
| auth       | public_key_path | 非空                        |
| auth       | issuer          | 非空                        |
