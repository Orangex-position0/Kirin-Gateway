# 传输安全

## TLS 终止

### 概述

网关在入口处对客户端连接执行 TLS 终止（TLS Termination），解密后的流量以明文形式转发给上游服务。客户端无需感知后端架构，只需与网关建立 HTTPS 连接。

### 配置方式

在 `config.yaml` 的 `server` 节下配置 TLS：

```yaml
server:
  listen: "0.0.0.0:6188"
  threads: 2
  tls:
    cert_path: "certs/server.crt"
    key_path: "certs/server.key"
```

- `tls` 为可选项，不配置则网关以 HTTP 明文模式监听
- `cert_path`：TLS 证书文件路径（PEM 格式）
- `key_path`：TLS 私钥文件路径（PEM 格式）

### 实现位置

| 文件 | 职责 |
|------|------|
| `src/config/types.rs` | `TlsConfig` 结构体定义，`ServerConfig.tls` 字段 |
| `src/main.rs` | 根据 `tls` 是否存在，调用 `add_tls()` 或 `add_tcp()` 选择监听模式 |
| `src/config/validation.rs` | 配置校验（复用 `validate_server`） |

### 启动流程

```text
main.rs
  └── 加载配置 → 检查 server.tls
        ├── 有 tls 配置 → lb_service.add_tls(listen, cert, key)
        │                 网关以 HTTPS 模式监听
        └── 无 tls 配置 → lb_service.add_tcp(listen)
                          网关以 HTTP 模式监听
```

### 限制与 TODO

- 当前仅支持单证书配置，不支持 SNI 多证书
- 不支持 mTLS（双向认证）
- 不支持运行时证书热更新（需重启网关）
- TLS 配置变更不在 `diff_update` 的增量更新范围内
